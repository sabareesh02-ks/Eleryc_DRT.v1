üåê OPTION 1: LOCAL NETWORK ACCESS (EASIEST - SAME OFFICE)
================================================================================

Perfect for: Colleagues in the same office/network who want to VIEW only

HOW IT WORKS:
-------------
When you run START_AUTOMATED_SERVER.bat, the Flask server is accessible
on your local network. Colleagues can access it from their browsers!

STEPS:
------

1. On YOUR Computer:
   ------------------
   a) Run START_AUTOMATED_SERVER.bat
   
   b) Find your IP address:
      - Open Command Prompt (cmd)
      - Type: ipconfig
      - Look for "IPv4 Address" under your network adapter
      - Example: 192.168.1.100

   c) Share this URL with colleagues:
      http://[YOUR_IP]:5000
      
      Example: http://192.168.1.100:5000

2. On COLLEAGUE's Computer:
   -------------------------
   a) Open browser
   
   b) Go to: http://[YOUR_IP]:5000
      (Replace with your actual IP address)
   
   c) They can now view and compare experiments!

IMPORTANT NOTES:
----------------
‚úÖ Works instantly - no installation needed for colleagues
‚úÖ You must keep your computer ON and server RUNNING
‚úÖ Colleagues can VIEW and COMPARE but changes happen on your computer
‚úÖ Only works on same network (same office/WiFi)
‚úÖ Your computer becomes the "server"

PROS:
- Instant access
- No setup for colleagues
- Real-time data

CONS:
- Your computer must stay on
- Only works on same network
- They can't use it when you're offline

üìù DETAILED INSTRUCTIONS FOR OPTION 1 (MOST COMMON)
================================================================================

STEP-BY-STEP: LOCAL NETWORK SHARING
------------------------------------

On YOUR Computer (Host):
------------------------

1. Find Your IP Address:
   
   Windows:
   --------
   a) Press Windows Key + R
   b) Type: cmd
   c) Press Enter
   d) Type: ipconfig
   e) Find "IPv4 Address" (e.g., 192.168.1.100)
   f) Write it down!

   Alternative (Easier):
   ---------------------
   a) Open Settings ‚Üí Network & Internet
   b) Click Properties
   c) Look for "IPv4 address"

2. Start Server:
   
   Double-click: START_AUTOMATED_SERVER.bat
   
   Important: Leave terminal window open!

3. Test It Yourself First:
   
   Open browser: http://localhost:5000
   Make sure it works!

4. Share with Colleagues:
   
   Tell them to open: http://[YOUR_IP]:5000
   
   Example: "Open your browser and go to http://192.168.1.100:5000"


On COLLEAGUE's Computer (Client):
----------------------------------

1. Make sure you're on the same network (same WiFi/office network)

2. Open ANY web browser (Chrome, Edge, Firefox)

3. Type in address bar: http://[HOST_IP]:5000
   (Replace [HOST_IP] with the IP address your colleague gave you)

4. Press Enter

5. You should see the Eleryc Experiment Viewer!

6. Use it like normal:
   - View individual experiments
   - Compare multiple experiments
   - All features work!


TROUBLESHOOTING OPTION 1:
--------------------------

Problem: "This site can't be reached"
Solution:
‚Ä¢ Check host computer's IP address is correct
‚Ä¢ Verify both computers are on same network
‚Ä¢ Check host computer's firewall:
  - Windows Firewall might block port 5000
  - Allow Python through firewall:
    1. Windows Security ‚Üí Firewall & network protection
    2. Allow an app through firewall
    3. Find Python ‚Üí Allow both Private and Public

Problem: "Connection refused"
Solution:
‚Ä¢ Make sure START_AUTOMATED_SERVER.bat is running on host
‚Ä¢ Check server shows "Server running at: http://localhost:5000"

Problem: Works on host but not for colleagues
Solution:
‚Ä¢ Firewall is likely blocking
‚Ä¢ Try temporarily disabling firewall to test
‚Ä¢ Or add exception for port 5000
